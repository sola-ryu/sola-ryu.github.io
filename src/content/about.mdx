navfolio is a small personal publication starter for people who want one quiet place for profile details, useful links, project notes, and long-form writing.

It treats a personal site less like a static resume and more like a readable operating surface: a notebook for public thoughts, a lightweight index for identity links, and a calm archive for writing over time.

## What This Space Is For

Use it to collect durable notes, article drafts, project records, and the small pieces of context that normally get scattered across profiles and repositories.

The default content is intentionally fictional. Replace the sample profile, links, GitHub username, and blog posts with your own material in `src/data/profile.ts`, `src/content/blog/`, and eventually `src/content/projects/`.

## How It Is Built

The project is Astro-first, content-driven, and static by default. Markdown and MDX content can be edited directly, rendered into article pages, and deployed to GitHub Pages, Vercel, Netlify, Cloudflare Pages, or any platform that can build an Astro project.

The visual language is intentionally restrained: soft structure, low-noise surfaces, readable typography, and small interactions that support reading instead of interrupting it.
